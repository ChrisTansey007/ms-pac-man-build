# Milestones

> **Milestones group related tasks into a release or delivery target.**

## Purpose

Milestones provide a lightweight planning layer above individual tasks. They help teams see progress toward a larger deliverable without losing the file-based task model.

## Files

- `milestone-template.md` — Canonical milestone template
- `../state/milestones.json` — Machine-readable milestone tracking state

## Rules

- Milestones should map to a small, coherent set of tasks.
- A milestone is not complete until all task dependencies and verification gates are satisfied.
- Keep milestone titles concise and outcome-focused.
