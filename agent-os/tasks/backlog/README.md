# Backlog

Tasks in this folder are **proposed but not yet approved** for work.

## Who Can Add Tasks Here

- Any worker performing task decomposition (typically Hermes or Claude).
- The human project owner.

## What Belongs Here

- Task files generated from goal decomposition.
- Proposed features, improvements, or fixes.
- Research tasks that need human approval before execution.

## How Tasks Leave Here

- **Approved:** Human moves the task to [`../ready/`](../ready/).
- **Rejected:** Human deletes the task file or moves it out.
- **Needs refinement:** Worker updates the task based on feedback.

## Rules

- Tasks in `backlog/` CANNOT be claimed by workers.
- Only the human can move tasks from `backlog/` to `ready/`.
- Workers should not modify tasks in `backlog/` without human direction.
