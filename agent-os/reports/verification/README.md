# Verification Reports

Task verification reports and evidence.

## Purpose

Verification reports document that a task has been independently verified against its acceptance criteria. They are the evidence that supports moving a task to `done/`.

## What Goes Here

- Verification reports for tasks in `review/` or `done/`.
- Test output files.
- Screenshots or recordings of UI changes.
- Logs demonstrating correct behavior.

## Naming

`[TASK-ID]-verification-[YYYY-MM-DD].md`
