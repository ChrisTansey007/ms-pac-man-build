# Checkpoints

> **Checkpoint files capture partial progress on long-running tasks without pretending the work is finished.**

## Purpose

Checkpoints let workers preserve intermediate progress, decisions, and open questions before a full handoff is ready.

## Files

- `handoff-checkpoint-template.md` — Canonical checkpoint template
- Checkpoint notes should live in this folder until the task is ready for a full handoff

## Rules

- A checkpoint is not a handoff.
- A checkpoint must say what is done, what remains, and what blocked the worker.
- Keep checkpoints short and factual.
