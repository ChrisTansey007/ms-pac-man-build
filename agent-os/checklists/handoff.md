# Handoff Checklist

> **Complete this checklist when writing a session handoff.**

## Template Completion

- [ ] Task ID included
- [ ] Worker name included
- [ ] Role performed included
- [ ] Capabilities used listed
- [ ] Date/time included
- [ ] Session status indicated (complete / in-progress / blocked)
- [ ] Summary of work written (2-4 sentences)

## Change Documentation

- [ ] All files changed listed with descriptions
- [ ] Commands run documented
- [ ] Tests run and results documented
- [ ] Evidence produced listed with locations

## Honesty

- [ ] What changed is accurately described
- [ ] What did NOT change is noted
- [ ] Known issues are honestly reported
- [ ] Risks are identified

## Forward Guidance

- [ ] Next recommended worker specified (with recommended role)
- [ ] Next steps are clear and actionable
- [ ] Verification notes included
- [ ] Reassignment notes included if applicable
- [ ] Enough context for a different worker to continue without private memory

## Placement

- [ ] Handoff saved to [`handoffs/active/`](../handoffs/active/)
- [ ] File name follows convention: `[TASK-ID]-[worker]-[YYYY-MM-DD].md`
- [ ] Task file updated to reference this handoff
