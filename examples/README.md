# Examples

> **Illustrative examples of ROWS artifacts. These are NOT active repo state.**

## What This Is

This directory contains sample artifacts showing the expected quality and format for ROWS deliverables. Workers should use these as reference for understanding what good output looks like.

## Important Warnings

1. **Examples are illustrative only.** They show fictional content for a fictional project.
2. **Actual work should happen in `agent-os/`.** Do not move examples into lifecycle folders.
3. **Examples are not active repo state.** They do not represent current project status.
4. **Adapt, don't copy.** Use examples to understand format and quality expectations, not as templates to fill in verbatim.

## Contents

| File | What It Shows |
|------|---------------|
| `sample-project-goal.md` | A filled-out `PROJECT_GOAL.md` for a fictional app |
| `sample-task.md` | A properly formatted task using the role/capability model |
| `sample-handoff.md` | A high-quality worker handoff |
| `sample-verification-report.md` | Verification evidence and pass/fail assessment |
| `sample-reassignment-record.md` | A worker switch from one worker to another |
| `sample-status-report.md` | A useful project status summary |

## The Fictional Example Project

All examples use "TaskFlow Lite" — a small task tracking app for solo builders. This is a fictional project used only for illustration.
